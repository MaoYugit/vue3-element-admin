# 入口文件 `src/main.ts` 分析

## 概述

`src/main.ts` 是 Vue3-Element-Admin 项目的应用入口文件，负责创建 Vue 应用实例并按顺序注册所有核心插件、全局组件、路由守卫及初始化服务，最后挂载到 DOM 节点上。

## 完整引导流程

```
createApp(App)
  ├─ setupDirective(app)      → 注册全局自定义指令 (v-hasPerm)
  ├─ setupI18n(app)           → 安装国际化插件 (vue-i18n)
  ├─ setupRouter(app)         → 安装路由 (vue-router, hash 模式)
  ├─ setupStore(app)          → 安装状态管理 (Pinia, 7 个 store)
  ├─ 全局注册 Element Plus 图标   → 所有图标作为全局组件
  ├─ setupPermissionGuard()   → 注册导航守卫 (权限/登录/动态路由)
  ├─ setupSse()               → 启动 SSE 长连接服务
  └─ app.mount("#app")        → 挂载应用
```

## 逐行分析

### 1. 创建 Vue 应用实例

```ts
import { createApp } from "vue";
import App from "./App.vue";
```

- `createApp` 是 Vue 3 提供的工厂函数，用于创建应用实例。
- `App.vue` 是根组件，其模板包含 `<el-config-provider>`、水印组件和 `<router-view>`。

### 2. 全局样式引入

```ts
import "element-plus/theme-chalk/dark/css-vars.css";
import "@/styles/index.scss";
import "uno.css";
import "animate.css";
```

| 导入路径 | 作用 |
|---|---|
| `element-plus/theme-chalk/dark/css-vars.css` | Element Plus 暗黑模式 CSS 变量 |
| `@/styles/index.scss` | 项目全局样式入口（主题变量、CSS Reset、布局样式、Element Plus 覆盖等） |
| `uno.css` | [UnoCSS](https://github.com/unocss/unocss) 即时原子化 CSS 引擎 |
| `animate.css` | 流行的 CSS 动画库 |

### 3. 功能模块导入

```ts
import { setupDirective } from "@/directives";
import { setupRouter } from "@/router";
import { setupStore } from "@/stores";
import { setupI18n } from "@/lang";
import * as ElementPlusIcons from "@element-plus/icons-vue";
import { setupPermissionGuard } from "@/router/guards/permission";
import { setupSse } from "@/composables";
```

| 导入模块 | 所属路径 | 说明 |
|---|---|---|
| `setupDirective` | `src/directives/index.ts` | 注册 `v-hasPerm` 权限指令 |
| `setupRouter` | `src/router/index.ts` | 安装 Vue Router（hash 模式 + 静态路由） |
| `setupStore` | `src/stores/index.ts` | 安装 Pinia（7 个 Store） |
| `setupI18n` | `src/lang/index.ts` | 安装 vue-i18n（中/英双语） |
| `ElementPlusIcons` | `@element-plus/icons-vue` | 所有 Element Plus 图标组件 |
| `setupPermissionGuard` | `src/router/guards/permission.ts` | 注册 beforeEach/afterEach 路由守卫 |
| `setupSse` | `src/composables/index.ts` | 启动 SSE 服务（字典同步 + 在线人数） |

### 4. 应用初始化顺序

#### 第 1 步：创建实例

```ts
const app = createApp(App);
```

#### 第 2 步：注册全局指令

```ts
setupDirective(app);
```

内部调用 `app.directive("hasPerm", ...)`，注册基于按钮级权限控制的 `v-hasPerm` 自定义指令。

#### 第 3 步：安装国际化

```ts
setupI18n(app);
```

内部调用 `app.use(i18n)`，注册 `vue-i18n` 实例，支持 `zh-cn` 和 `en` 两种语言。

#### 第 4 步：安装路由

```ts
setupRouter(app);
```

内部调用 `app.use(router)`，注册 Vue Router 实例（hash 模式），包含登录页、Dashboard、404、个人中心等常量路由。

#### 第 5 步：安装状态管理

```ts
setupStore(app);
```

内部调用 `app.use(pinia)`，注册 Pinia 实例。包含以下 Store：

| Store ID | 文件名 | 职责 |
|---|---|---|
| `app` | `stores/app.ts` | 设备类型、侧边栏状态、语言、组件尺寸、全屏 |
| `dict` | `stores/dict.ts` | 字典数据缓存 |
| `permission` | `stores/permission.ts` | 动态路由生成 |
| `setting` | `stores/settings.ts` | 系统设置（主题、布局、水印等） |
| `tagsView` | `stores/tags-view.ts` | 标签页管理 |
| `tenant` | `stores/tenant.ts` | 多租户支持 |
| `user` | `stores/user.ts` | 用户认证状态（登录/登出/Token 刷新） |

#### 第 6 步：全局注册 Element Plus 图标

```ts
Object.entries(ElementPlusIcons).forEach(([name, comp]) => app.component(name, comp));
```

将 `@element-plus/icons-vue` 导出的所有图标组件以 `<el-icon-xxx>` 或直接 `<xxx>` 的形式注册为全局组件，无需在每个页面单独 import。

#### 第 7 步：注册导航守卫

```ts
setupPermissionGuard();
```

安装 `router.beforeEach` 全局前置守卫，处理：

- **未登录** → 白名单路径（如 `/login`）允许访问，否则重定向到 `/login?redirect=...`
- **已登录访问 `/login`** → 自动重定向到 `/`
- **动态路由生成** → 首次加载时调用 `permissionStore.generateRoutes()`，将后端返回的菜单数据转为 Vue Router 路由并动态添加
- **404 处理** → 未匹配的路由重定向到 404 页面
- **动态标题** → 支持从 URL 参数中读取页面标题

同时安装 `router.afterEach`，停止 NProgress 加载条。

#### 第 8 步：启动 SSE 服务

```ts
setupSse();
```

启动两个基于 Server-Sent Events 的后台服务：

- **字典同步（`useDictSync`）**：实时同步服务端字典变更
- **在线人数（`useOnlineCount`）**：实时获取当前在线用户数

SSE 底层使用 Fetch API + ReadableStream 实现，支持 Token 认证、指数退避重连（5s~120s，最多 10 次）。

#### 第 9 步：挂载应用

```ts
app.mount("#app");
```

将 Vue 应用挂载到 HTML 中 `id="app"` 的 DOM 节点上。

## 设计特点

### 1. 模块化的初始化函数

每个核心功能（指令、国际化、路由、状态管理）都有独立的 `setupXxx(app)` 函数，职责单一、易于测试和维护。

### 2. 插件的安装顺序

安装顺序存在隐含依赖关系：

- `setupRouter` 必须先于 `setupPermissionGuard`（守卫依赖路由实例）
- `setupStore` 必须先于 `setupPermissionGuard` 和 `setupSse`（守卫和 SSE 依赖 Pinia Store 中的用户/权限状态）
- `setupI18n` 建议先于 `setupRouter`（路由配置可能使用国际化）

### 3. 按需分离的初始化和启动

- 同步操作（指令、插件注册）在挂载前完成
- 异步/后台操作（SSE 连接）在挂载后才触发（但代码中是在挂载前注册，挂载后连接）
- 路由守卫在插件安装完成后、挂载前注册，确保首次导航即生效

### 4. 零配置图标注册

通过 `Object.entries` 遍历 `@element-plus/icons-vue` 的所有导出，自动注册为全局组件，无需手动维护图标列表。

## 与 `index.html` 的关联

`public/index.html`（或项目根目录下的 `index.html`）中包含 `<div id="app"></div>` 作为挂载点：

```html
<div id="app"></div>
<script type="module" src="/src/main.ts"></script>
```

Vite 构建时会自动将 `main.ts` 作为入口进行打包。
