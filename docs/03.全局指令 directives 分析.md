# 全局指令模块 `src/directives` 分析

## 目录结构

```
src/directives/
├── index.ts              # 入口，全局注册所有指令
└── permission/
    └── index.ts          # 权限指令实现（v-hasPerm / v-hasRole）
```

## 入口文件 `index.ts`

```ts
export function setupDirective(app: App<Element>) {
  app.directive("hasPerm", hasPerm);
}
```

- 导出 `setupDirective(app)` 函数，在 `main.ts` 中被调用
- 目前只全局注册了 `v-hasPerm` 指令
- `v-hasRole` 虽然已实现，但未在此注册，可手动按需导入使用

## 权限指令 `permission/index.ts`

该文件定义了两个 Vue 自定义指令：`hasPerm`（按钮权限）和 `hasRole`（角色权限），均基于 Vue 3 的 `Directive` 接口实现。

### 通用行为

- 在指令的 `mounted` 钩子中执行权限校验
- 校验不通过时，直接将 DOM 元素从父节点移除（`el.parentNode.removeChild(el)`）
- 支持传入 `string` 或 `string[]` 两种格式的权限值

### `v-hasPerm` — 按钮权限指令

#### 注册名称
`hasPerm` → 模板中使用 `v-hasPerm`

#### 作用
根据当前用户的 `perms`（权限标识列表）控制元素的显示/隐藏。

#### 用法

```vue
<!-- 单个权限标识 -->
<el-button v-hasPerm="'sys:user:create'">新增</el-button>

<!-- 多个权限标识（满足任一即可） -->
<el-button v-hasPerm="['sys:user:create', 'sys:user:update']">操作</el-button>
```

#### 校验逻辑

```
mounted(el, binding)
  ├─ 校验 binding.value 合法性（必须为 string 或 array）
  ├─ 获取当前用户信息中的 roles 和 perms
  ├─ 超级管理员豁免 ──── 如果 roles 包含 ROLE_ROOT → 直接放行
  ├─ 通配符豁免 ──────── 如果 requiredPerms 包含 "*:*:*" → 直接放行
  ├─ 权限匹配 ────────── array 模式：some() 任一匹配即可
  │                      string 模式：perms.includes() 精确匹配
  └─ 不匹配 ──────────── 从 DOM 中移除该元素
```

### `v-hasRole` — 角色权限指令

#### 注册名称
`hasRole` → 模板中使用 `v-hasRole`（需手动注册或直接导入后局部使用）

#### 作用
根据当前用户的 `roles`（角色标识列表）控制元素的显示/隐藏。

#### 用法

```vue
<template v-if="hasRole">
  <el-button v-hasRole="'ADMIN'">管理员操作</el-button>
</template>

<!-- 多个角色满足任一即可 -->
<el-button v-hasRole="['ADMIN', 'TEST']">测试操作</el-button>
```

#### 校验逻辑

```
mounted(el, binding)
  ├─ 校验 binding.value 合法性
  ├─ 获取当前用户的 roles
  ├─ 角色匹配 ──── array 模式：some() 任一匹配
  │                 string 模式：roles.includes() 精确匹配
  └─ 不匹配 ────── 从 DOM 中移除该元素
```

> 与 `v-hasPerm` 的区别：`v-hasRole` **没有**超级管理员豁免逻辑，仅做角色字符串匹配。

## 与用户 Store 的交互

两个指令都通过 `useUserStore()` 获取当前用户信息：

```ts
const { roles, perms } = useUserStore().userInfo;
```

`userInfo` 的数据结构（来自 `src/stores/user.ts`）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `roles` | `string[]` | 当前用户的角色列表，如 `["ROOT"]` |
| `perms` | `string[]` | 当前用户的权限标识列表，如 `["sys:user:create", "sys:user:delete"]` |

常量 `ROLE_ROOT` 定义在 `src/constants/index.ts`：

```ts
export const ROLE_ROOT = "ROOT";
```

---

## 设计特点

### 1. "移除元素"而非"隐藏"

权限校验失败时直接调用 `removeChild` 从 DOM 中移除元素，而非使用 `v-if` / `v-show` 控制显隐。这样做的好处是：

- 敏感操作按钮彻底不在 DOM 中存在，避免被样式覆盖或脚本绕过
- 减少了 DOM 节点数量

### 2. 单文件双指令

`hasPerm` 和 `hasRole` 实现在同一文件中，逻辑高度相似，便于统一维护。未来可整体迁移到组合式 API 风格。

### 3. 严格的输入校验

在 `mounted` 钩子开头对 `binding.value` 进行类型校验，非法输入直接 `throw Error`，在开发阶段即可发现问题。

### 4. 超级管理员默认放行

`v-hasPerm` 中对 `ROLE_ROOT` 角色做了硬编码豁免，避免每次新增功能都需要为超级管理员配置权限。

## 扩展指南

### 注册更多指令

在 `src/directives/index.ts` 中追加即可：

```ts
import { hasPerm, hasRole } from "./permission";

export function setupDirective(app: App<Element>) {
  app.directive("hasPerm", hasPerm);
  app.directive("hasRole", hasRole); // 注册角色指令
}
```

### 局部使用（不全局注册）

```vue
<script setup lang="ts">
import { hasRole } from "@/directives/permission";
const vHasRole = hasRole;
</script>
```
