# 路由模块 `src/router` 分析

## 目录结构

```
src/router/
├── index.ts              # 路由实例创建、静态路由定义、全局注册
└── guards/
    └── permission.ts     # 路由权限守卫（登录验证 / 动态路由 / 404 检测）
```

---

## 路由实例 `index.ts`

### 路由模式

使用 `createWebHashHistory()` 创建 hash 模式路由。URL 格式形如 `http://localhost/#/dashboard`。

Hash 模式兼容性更好，无需服务端做路由回退配置。

### 静态路由（常量路由）

`constantRoutes` 数组定义了所有无需登录即可访问或应用内置的基础路由：

| 路径 | 组件 | 说明 |
|---|---|---|
| `/redirect/:path(.*)` | `views/redirect.vue` | 全局重定向处理 |
| `/login` | `views/login/index.vue` | 登录页 |
| `/dashboard` | `views/dashboard/index.vue` | 首页仪表盘（`affix: true` 固定标签页，`keepAlive: true` 缓存） |
| `/401` | `views/error/401.vue` | 无权限页 |
| `/404` | `views/error/404.vue` | 页面不存在 |
| `/profile` | `views/profile/index.vue` | 个人中心（`hidden: true`） |
| `/profile/notice` | `views/profile/notice/index.vue` | 我的通知（`hidden: true`） |
| `/detail/:id(\d+)` | `views/demo/detail.vue` | Demo 详情（带参数校验，`keepAlive: true`） |

其中 `Dashboard`、`Profile`、`MyNotice`、`DemoDetail` 在 Layout 布局下渲染，其余为独立页面。

### Layout 组件

```ts
export const Layout = () => import("@/layouts/index.vue");
```

使用动态 `import()` 实现懒加载。`Layout` 导出为命名导出，供后续动态路由生成时引用。

### 路由元信息（meta）

| 属性 | 类型 | 说明 |
|---|---|---|
| `title` | `string` | 页面标题，用于面包屑/标签页/浏览器标题 |
| `icon` | `string` | 侧边栏菜单图标名称 |
| `hidden` | `boolean` | 是否在侧边栏中隐藏 |
| `affix` | `boolean` | 是否固定到标签页（不可关闭） |
| `keepAlive` | `boolean` | 是否用 `<keep-alive>` 缓存组件 |

### 滚动行为

```ts
scrollBehavior: () => ({ left: 0, top: 0 })
```

每次路由切换后自动将页面滚动到顶部。

### 注册函数

```ts
export function setupRouter(app: App<Element>) {
  app.use(router);
}
```

## 路由权限守卫 `guards/permission.ts`

### 整体流程

```
router.beforeEach
  │
  ├─ isLoggedIn == false
  │    ├─ 白名单路径 (/login) ─── 放行
  │    └─ 其他路径 ────────────── 重定向 /login?redirect=xxx
  │
  ├─ to.path == "/login" (已登录) ─ 重定向到 /
  │
  ├─ isRouteGenerated == false
  │    ├─ 获取用户信息（如 roles 为空）
  │    ├─ 初始化多租户上下文（可选）
  │    ├─ 调用 permissionStore.generateRoutes()
  │    ├─ 动态添加路由 → router.addRoute()
  │    └─ replace 当前导航（重走守卫）
  │
  ├─ to.matched.length == 0
  │    ├─ 来自 /login → 回退首页
  │    └─ 其他 → 跳转 /404
  │
  ├─ 动态标题 (to.params.title / to.query.title)
  │
  └─ catch(error) → resetAllState → 重定向 /login


router.afterEach
  └─ NProgress.done()
```

### 守卫详情

#### 1. 登录验证

```ts
const isLoggedIn = useUserStore().isLoggedIn();
```

未登录时：
- 访问 `/login` → 放行
- 其他路径 → 追加 `?redirect=` 参数后跳转 `/login`，登录后自动回到原页面

#### 2. 动态路由生成

核心逻辑：首次访问时，调用 `permissionStore.generateRoutes()` 从后端获取菜单配置并添加为动态路由。

- 条件：`permissionStore.isRouteGenerated === false`
- 前置：若 `userStore.userInfo.roles` 为空，先调 `getUserInfo()` 获取角色信息
- 可选：若 `VITE_APP_TENANT_ENABLED=true`，加载租户列表
- 动作：逐个 `router.addRoute(route)` 添加动态路由
- 完成：`return { ...to, replace: true }` 重走当前导航以匹配新路由

#### 3. 404 检测

```ts
if (to.matched.length === 0) { ... }
```

- 从 `/login` 跳转过来的无效路径 → 回退首页（不同用户角色权限不同，登录后路由可能已变化）
- 其他情况 → 显示 404 页面

#### 4. 动态标题

支持从路由参数 `params.title` 或 `query.title` 中读取并设置 `to.meta.title`，用于详情页等动态页面。

#### 5. 错误处理

```ts
catch (error) {
  await useUserStore().resetAllState();
  return "/login";
}
```

任何守卫执行过程中出现异常，重置用户所有状态（清除权限、字典缓存、标签页、SSE 连接）并跳转回登录页。

#### 6. 多租户上下文

```ts
async function initTenantContext(): Promise<void> {
  if (!isTenantEnabled()) return;
  await useTenantStoreHook().loadTenant();
}
```

通过 `VITE_APP_TENANT_ENABLED` 环境变量控制。启用时自动加载当前用户的租户列表；静默失败，不影响主流程。

## 动态路由模块的关联

`src/router/index.ts` 中定义的是静态路由，`src/stores/permission.ts` 负责动态路由的生成。两者的关系：

```
静态路由 (constantRoutes)   动态路由 (permissionStore.generateRoutes)
        │                              │
        ├── /redirect                  ├── 从后端 API 获取菜单配置
        ├── /login                     ├── 转换为 RouteRecordRaw 格式
        ├── / (Layout + children)      └── 通过 router.addRoute() 注入
        ├── /401
        └── /404
```

动态路由的生成依赖于 `usePermissionStore`，该 Store 调用 `MenuAPI.getRoutes()` 获取后端菜单树，递归转换为符合 Vue Router 格式的路由记录。

## 设计特点

### 1. 静态与动态路由分离

- 静态路由：登录页、404/401、首页等核心页面，应用启动即注册
- 动态路由：后端菜单树，首次导航时异步生成，不同角色看到不同菜单

### 2. replace 重走守卫

动态路由添加完毕后，通过 `return { ...to, replace: true }` 重新触发 `beforeEach`，使新添加的路由立即生效，同时替换掉当前的历史记录，避免用户回退时再次触发路由生成。

### 3. 白名单机制

只有 `/login` 在白名单中，其余所有路径都需要登录验证，确保未认证用户只能看到登录页。

### 4. 错误即登出

守卫中出现任何异常（如 token 过期、网络错误）都会重置所有状态并跳转回登录页，避免应用处于不一致的状态。

### 5. NProgress 集成

`beforeEach` 中启动加载条，`afterEach` 中结束，配合页面异步加载提供视觉反馈。
