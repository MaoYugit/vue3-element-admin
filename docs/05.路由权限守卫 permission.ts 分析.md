# 路由权限守卫 `guards/permission.ts` 分析

## 文件定位

`src/router/guards/permission.ts` 是应用的核心导航守卫模块，在 `main.ts` 中通过 `setupPermissionGuard()` 调用，注册 `router.beforeEach` 和 `router.afterEach` 全局钩子，处理完整的导航生命周期。

---

## 导出

```ts
export function setupPermissionGuard()
```

无参数、无返回值。内部通过闭包引用 `router` 实例来注册守卫。

---

## 完整执行流程

```
router.beforeEach(to, from)
  │
  ├── 1. 启动 NProgress
  │
  ├── 2. 未登录处理
  │     ├─ to.path 在白名单 ["/login"] 中 ─── 放行 (return undefined)
  │     └─ 否则 ─────────────────────────── 重定向 /login?redirect=xxx
  │
  ├── 3. 已登录 + 访问 /login ── 重定向到 /
  │
  ├── 4. 首次加载（动态路由未生成）
  │     ├─ 获取用户信息（若 roles 为空）
  │     ├─ 初始化租户上下文（若启用）
  │     ├─ permissionStore.generateRoutes() → 获取并转换动态路由
  │     ├─ 逐个 router.addRoute() 添加
  │     └─ return { ...to, replace: true } 重走导航
  │
  ├── 5. 404 检测（to.matched.length === 0）
  │     ├─ 来自 /login → 回退首页
  │     └─ 其他 → 跳转 /404
  │
  ├── 6. 动态标题（从 params/query 读取 title）
  │
  └── catch(error) → resetAllState → 重定向 /login


router.afterEach
  └── NProgress.done()
```

---

## 逐段分析

### 1. 白名单

```ts
const whiteList = ["/login"];
```

未登录状态下唯一可访问的路由。若需添加其他公开页面（如注册页、忘记密码），在此数组中追加即可。

### 2. 未登录处理

```ts
if (!isLoggedIn) {
  if (whiteList.includes(to.path)) {
    return;
  }
  NProgress.done();
  return `/login?redirect=${encodeURIComponent(to.fullPath)}`;
}
```

- 白名单路径：直接放行
- 非白名单路径：停止进度条，重定向到登录页，并在 query 中附加 `redirect` 参数，登录后可自动回到原目标页

### 3. 已登录访问登录页

```ts
if (to.path === "/login") {
  return { path: "/" };
}
```

防止已登录用户再次看到登录页。

### 4. 用户信息懒获取

```ts
if (!userStore.userInfo?.roles?.length) {
  await userStore.getUserInfo();
}
```

- 首次访问时，若用户信息尚未加载（页面刷新后 Pinia 状态丢失），先调接口获取用户信息
- `getUserInfo()` 内部会填充 `userInfo.roles` 和 `userInfo.perms`，这是后续权限判断的依据

### 5. 动态路由生成

```ts
if (!permissionStore.isRouteGenerated) {
  // ... 获取用户信息、初始化租户 ...
  const dynamicRoutes = await permissionStore.generateRoutes();
  dynamicRoutes.forEach((route: RouteRecordRaw) => {
    router.addRoute(route);
  });
  return { ...to, replace: true };
}
```

- `isRouteGenerated` 初始为 `false`，生成后置为 `true`
- `generateRoutes()` 调用后端菜单 API，将返回的菜单树递归转换为 `RouteRecordRaw`
- 每个动态路由通过 `router.addRoute()` 添加到现有路由表
- `return { ...to, replace: true }` 使导航重走，让新路由生效

### 6. 404 检测

```ts
if (to.matched.length === 0) {
  if (_from.path === "/login") {
    return { path: "/", replace: true };
  }
  return "/404";
}
```

- `to.matched` 为空 → 没有路由匹配
- 如果来自登录页（可能是切换账号后权限变化），回退到首页而非显示 404
- 其他情况正常跳转 404

### 7. 动态标题

```ts
const title = (to.params.title as string) || (to.query.title as string);
if (title) {
  to.meta.title = title;
}
```

支持两种方式传递标题：
- `router.push({ name: "Detail", params: { title: "xxx" } })`
- `<router-link :to="{ query: { title: 'xxx' } }">`

设置后，面包屑、浏览器标题等使用 `route.meta.title` 的地方会自动反映该值。

### 8. 异常捕获

```ts
catch (error) {
  console.error("Route guard error:", error);
  await useUserStore().resetAllState();
  NProgress.done();
  return "/login";
}
```

捕获所有守卫执行中的异常（网络错误、token 过期、API 异常等）：

- `resetAllState()` 清理所有用户相关状态（权限 Store 的 `isRouteGenerated` 重置、字典缓存、标签页、SSE 连接）
- 跳转回登录页，保证应用处于干净的未登录状态

### 9. NProgress

```ts
router.beforeEach → NProgress.start()
router.afterEach  → NProgress.done()
```

- `beforeEach` 启动进度条（包括重定向场景）
- `afterEach` 结束进度条
- 未登录跳转时手动调用 `NProgress.done()` 避免进度条卡住

## 辅助函数

### `initTenantContext()`

```ts
async function initTenantContext(): Promise<void> {
  if (!isTenantEnabled()) return;
  try {
    await useTenantStoreHook().loadTenant();
  } catch {
    // 静默失败
  }
}
```

- 受 `VITE_APP_TENANT_ENABLED` 环境变量控制
- 启用时从后端加载当前用户的租户列表
- 失败时静默处理，不影响主登录流程

## 边缘场景处理

| 场景 | 处理方式 |
|---|---|
| 页面刷新导致 Pinia 状态丢失 | `isRouteGenerated` 恢复为 `false`，重新生成路由 |
| 不同角色切换账号 | 来自 `/login` 时无效路径回退首页而非 404 |
| Token 过期/网络中断 | `catch` 中 `resetAllState()` 并跳转登录页 |
| 手动输入不存在的路由 | `to.matched.length === 0` → 404 |
| 从登录页携带 redirect 回来 | 登录后 `redirect` 参数指向的路径被正常导航 |
| 多租户 API 失败 | `catch` 静默，不阻塞登录流程 |
