# 多标签页（Tags View）系统学习大纲

## 一、先修知识

学习本模块前，建议先了解以下 Vue 3 基础知识：

- Vue 3 Composition API（`ref`, `reactive`, `computed`, `watch` 等）
- Pinia 状态管理（`defineStore`、`storeToRefs`）
- Vue Router（路由定义、导航守卫、`useRoute`/`useRouter`）
- Vue 内置组件 `<keep-alive>` 的 `include` 属性
- Element Plus 组件（`el-scrollbar`, `el-dropdown`, `el-icon`）

## 二、学习路线（由浅入深，共 6 步）

### 第 1 步：理解数据模型 — 从 Store 入口

**目标**：掌握标签页的数据结构和核心操作

**阅读文件**：`src/stores/tags-view.ts`

**要点**：

1. `TagView` 接口定义 — 一个标签包含哪些信息（name, title, path, fullPath, icon, affix, keepAlive, query）
2. 两个核心状态：
   - `visitedViews` — 已打开的标签列表（决定 UI 渲染哪些标签）
   - `cachedViews` — 需要 keep-alive 缓存的页面列表（决定哪些组件实例被缓存）
3. 核心操作分类：
   - **增**：`addView` / `addVisitedView` / `addCachedView`
   - **删**：`delView` / `delOtherViews` / `delLeftViews` / `delRightViews` / `delAllViews`
   - **改**：`updateVisitedView` / `updateTagName`
   - **查**：`isActive`
   - **跳转**：`toLastView`（关闭当前标签后自动跳转）

**思考题**：
- 为什么 `addVisitedView` 要跳过 `/redirect` 路径和外部链接？
- `affix` 属性起什么作用？

---

### 第 2 步：理解驱动的源头 — 路由变化 → 添加标签

**目标**：了解标签是如何"自动"出现的

**阅读文件**：`src/layouts/components/LayoutTagsView.vue`

**要点**：

1. 第 456-463 行 — `watch(route, ...)`：
   - 监听当前路由对象的变化
   - 每次路由切换时调用 `addCurrentTag()` 和 `updateCurrentTag()`
   - `{ immediate: true }` 保证首次加载也触发

2. `addCurrentTag()` 函数（第 292-306 行）：
   - 从 `route.meta` 中读取 title、icon、affix、keepAlive 等
   - 调用 `tagsViewStore.addView()` 将当前路由存入 store
   - 这是整个标签系统**最核心的入口**

3. 补充：`initAffixTags()` 在 `onMounted` 时执行（第 465-467 行）：
   - 遍历所有路由配置，找到 `meta.affix: true` 的固定标签（如 Dashboard）
   - 在页面初始化时就添加到标签栏

**思考题**：
- 如果用户直接在浏览器地址栏输入 URL 回车，标签是否会出现？
- `updateCurrentTag` 解决了什么问题（提示：对比 `route.path` 和 `route.fullPath`）？

### 第 3 步：理解可视化渲染 — 标签栏组件

**目标**：了解标签栏如何展示、交互

**阅读文件**：`src/layouts/components/LayoutTagsView.vue`（模板部分 1-132 行）

**要点**：

1. **模板结构**：
   - `<el-scrollbar>` 包裹水平滚动的标签列表
   - `v-for="tag in visitedViews"` 循环渲染每个标签
   - 每个标签包含：图标 → 标题 → 关闭按钮（affix 标签无关闭按钮）
   - 右侧操作区：刷新按钮、全屏按钮、下拉菜单

2. **交互功能**：
   - 左键点击标签 → `openTag()` → `router.push`
   - 鼠标中键点击 → 关闭标签
   - 右键点击 → 自定义上下文菜单（Teleport 到 body）
   - 滚轮滚动 → 水平滚动标签栏
   - 下拉菜单：刷新当前/关闭当前/关闭其它/关闭左侧/关闭右侧/关闭所有

3. **两种样式**：Card 和 Line，通过 `settingsStore.tagsViewStyle` 切换

4. **上下文菜单**的位置自适应（第 188-207 行）：
   - 计算菜单宽高，防止溢出视口

**思考题**：
- 为什么关闭按钮使用 `@click.stop`？
- 上下文菜单位置为什么用 `Teleport to="body"`？

### 第 4 步：理解缓存联动 — Keep-Alive 机制

**目标**：理解标签如何与页面缓存配合

**阅读文件**：`src/layouts/components/LayoutMain.vue`

**要点**：

1. 第 25 行 — `const { cachedViews } = toRefs(useTagsViewStore())`
   - 直接从 store 获取 `cachedViews` 列表

2. 第 6 行 — `<keep-alive :include="cachedViews">`
   - `include` 绑定到 store 中的缓存列表
   - 只有列表中的页面才会被缓存（切换时保持状态）

3. 第 29-58 行 — `currentComponent()` 函数：
   - 使用一个 `wrapperMap` 来为每个路由创建一个包装组件
   - 包装组件的 `name` 设为其 `fullPath`，供 keep-alive 匹配
   - 实现了基于路由 fullPath 的 keep-alive 缓存

4. 刷新功能逆向流程（LayoutTagsView.vue 第 372-379 行）：
   - 先从 `cachedViews` 中移除该页面
   - 跳转到 `/redirect` 路由，再立即重定向回来
   - 实现了页面重新渲染（类似刷新）

**思考题**：
- 如果关闭一个标签页，它对应的缓存会发生什么？
- `wrapperMap` 为什么要限制最大 100 个？

### 第 5 步：理解关闭和跳转逻辑

**目标**：掌握删除标签后的善后处理

**阅读文件**：`src/stores/tags-view.ts` 中的 `delView`、`toLastView`、`closeCurrentView`

**要点**：

1. `delView()`（第 141-150 行）：
   - 同时从 `visitedViews` 和 `cachedViews` 中删除
   - 返回删除后的新列表

2. `closeSelectedTag()`（LayoutTagsView.vue 第 381-389 行）：
   - 调用 `delView` 后，判断关闭的是否为当前激活标签
   - 如果是，调用 `toLastView()` 跳转到剩余最后一个标签

3. `toLastView()`（第 266-280 行）：
   - 取 `visitedViews` 最后一个作为目标
   - 如果没有标签了，跳转到首页 `/`
   - 如果是名为 "Dashboard" 的标签，用 `/redirect` 方式强制刷新

4. 方向性删除（`delLeftViews` / `delRightViews`）：
   - 找到当前标签的索引位置
   - 保留索引之前（或之后）的标签
   - 被删除的标签对应的缓存也要清除

**思考题**：
- 关闭所有标签后发生了什么？
- 为什么关闭最后一个标签后要特殊处理 Dashboard？

### 第 6 步：串联全局 — 布局中的集成方式

**目标**：理解组件在整体布局中的位置

**阅读文件**：`src/layouts/modes/LeftLayout.vue`（以及 MixLayout、TopLayout、DoubleLayout）

**要点**：

1. 布局结构（LeftLayout.vue 第 23-26 行）：
   ```
   LayoutNavbar
   LayoutTagsView  ← 标签栏（可开关）
   LayoutMain       ← 主要内容区（含 keep-alive）
   ```

2. 可开关控制：
   - `<LayoutTagsView v-if="showTagsView" />` 通过 `settingsStore.showTagsView` 控制
   - 用户可以在设置面板中关闭整个标签栏

3. 所有布局模式（Left/Mix/Top/Double）都包含相同的三段式结构：
   - 导航栏 + 标签栏 + 内容区

4. 高度自适应（LayoutMain.vue 第 61-67 行）：
   - 显示标签栏时：`100vh - navbar高度 - tags-view高度`
   - 隐藏标签栏时：`100vh - navbar高度`

**思考题**：
- 为什么标签栏放在布局组件中，而不是全局注册？
- 如果要在新布局中使用标签功能，需要怎么做？
