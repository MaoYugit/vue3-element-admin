# tags-view Store 详解

> 文件路径：`src/stores/tags-view.ts`

## 第一部分：接口定义与状态声明

### 1.1 TagView 接口（第 7-16 行）

```typescript
export interface TagView {
  name: string;        // 路由名称（对应路由的 name 字段）
  title: string;       // 标签显示标题（对应 route.meta.title）
  path: string;        // 路由路径（如 /dashboard）
  fullPath: string;    // 完整路径（带 query 参数，如 /detail?id=1）
  icon?: string;       // 图标名称（可选）
  affix?: boolean;     // 是否固定标签（不可关闭，如首页 Dashboard）
  keepAlive?: boolean; // 是否需要 keep-alive 缓存
  query?: LocationQuery; // 路由 query 参数
}
```

每个标签就是一个 `TagView` 对象。`path` 和 `fullPath` 的区别：`/detail?id=1` → path=`/detail`，fullPath=`/detail?id=1`。`affix: true` 的标签固定在标签栏，没有关闭按钮。

### 1.2 两个返回接口（第 21-31 行）

```typescript
export interface TagsViewResult {
  visitedViews: TagView[];
  cachedViews: string[];
}

export interface DirectionalTagsViewResult {
  visitedViews: TagView[];
}
```

- `TagsViewResult`：同时返回两个列表（普通删除、关闭其它、关闭全部用）
- `DirectionalTagsViewResult`：只返回 visitedViews（方向性删除用）。方向性删除在 filter 内部已通过 `cachedViews.splice` 直接修改了响应式数据，调用方只需 visitedViews 来判断跳转

### 1.3 两个核心状态（第 37-40 行）

```typescript
export const useTagsViewStore = defineStore("tagsView", () => {
  const visitedViews = ref<TagView[]>([]);
  const cachedViews = ref<string[]>([]);
  const router = useRouter();
  const route = useRoute();
```

整个 store 的全部状态只有两个数组：

- `visitedViews`：已打开的标签列表，决定 UI 渲染哪些标签
- `cachedViews`：需要 keep-alive 缓存的页面 fullPath 列表（`<keep-alive :include>` 要求字符串数组）

`useRouter()` 和 `useRoute()` 由 `unplugin-auto-import` 自动导入（配置在 `vite.config.ts` 第 61 行：`imports: ["vue", "pinia", "vue-router", ...]`），无需手动 import。

## 第二部分：添加标签

### 2.1 addVisitedView（第 45-60 行）

```typescript
function addVisitedView(view: TagView) {
  // 条件1：跳过 redirect 路径和外部链接
  if (view.path.startsWith("/redirect") || isExternal(view.path) || isExternal(view.fullPath)) {
    return;
  }
  // 条件2：去重 — 按 path 匹配，同一路由不会重复添加
  if (visitedViews.value.some((v) => v.path === view.path)) {
    return;
  }
  // 条件3：affix 插开头，普通插末尾
  if (view.affix) {
    visitedViews.value.unshift(view);
  } else {
    visitedViews.value.push(view);
  }
}
```

三个过滤条件：
1. **跳过中转和外部链接**：`/redirect` 路径是中转页，不应出现在标签栏；外部链接用新窗口打开，不产生标签
2. **去重**：按 `path` 匹配（`some()` 遍历）
3. **affix 处理**：`unshift` 插入数组开头（UI 最左侧），`push` 追加到末尾

### 2.2 addCachedView（第 65-75 行）

```typescript
function addCachedView({ fullPath, keepAlive }: TagView) {
  if (cachedViews.value.includes(fullPath)) return;
  if (keepAlive) cachedViews.value.push(fullPath);
}
```

只做两件事：去重 + 只有 `keepAlive: true` 才添加。

### 2.3 addView（第 156-159 行）

```typescript
function addView(view: TagView) {
  addVisitedView(view);
  addCachedView(view);
}
```

最常用的对外接口，一次调用完成"添加到标签栏 + 添加到缓存"。

## 第三部分：删除标签

### 3.1 单标签删除

```typescript
async function delVisitedView(view: TagView) {
  for (const [i, v] of visitedViews.value.entries()) {
    if (v.path === view.path) {
      visitedViews.value.splice(i, 1);
      break;
    }
  }
  return [...visitedViews.value];
}

async function delCachedView(view: TagView) {
  const { fullPath } = view;
  const index = cachedViews.value.indexOf(fullPath);
  if (index > -1) {
    cachedViews.value.splice(index, 1);
  }
  return [...cachedViews.value];
}
```

`delVisitedView` 按 `path` 匹配，`delCachedView` 按 `fullPath` 匹配。**重要变化**：从之前的 `new Promise(resolve => {... resolve()})` 改为了 `async function`，async 函数会自动返回 Promise，代码更简洁。

```typescript
async function delView(view: TagView): Promise<TagsViewResult> {
  await Promise.all([delVisitedView(view), delCachedView(view)]);
  return {
    visitedViews: [...visitedViews.value],
    cachedViews: [...cachedViews.value],
  };
}
```

**另一个重要变化**：`delView` 改用 `Promise.all` 并行执行两个删除操作，而不是之前的串行调用。

### 3.2 关闭其他

```typescript
async function delOtherVisitedViews(view: TagView) {
  visitedViews.value = visitedViews.value.filter((v) => {
    return v?.affix || v.path === view.path;
  });
  return [...visitedViews.value];
}

async function delOtherCachedViews(view: TagView) {
  const { fullPath } = view;
  const index = cachedViews.value.indexOf(fullPath);
  if (index > -1) {
    cachedViews.value = cachedViews.value.slice(index, index + 1);
  } else {
    cachedViews.value = [];
  }
  return [...cachedViews.value];
}
```

逻辑：只保留 `affix` 标签 + 目标标签。缓存列表只保留目标标签的缓存。

```typescript
async function delOtherViews(view: TagView): Promise<TagsViewResult> {
  await Promise.all([delOtherVisitedViews(view), delOtherCachedViews(view)]);
  return {
    visitedViews: [...visitedViews.value],
    cachedViews: [...cachedViews.value],
  };
}
```

### 3.3 关闭全部

```typescript
async function delAllViews(): Promise<TagsViewResult> {
  const affixTags = visitedViews.value.filter((tag) => tag?.affix);
  visitedViews.value = affixTags;
  cachedViews.value = [];
  return {
    visitedViews: [...visitedViews.value],
    cachedViews: [...cachedViews.value],
  };
}
```

visitedViews 只保留 affix，cachedViews 全部清空。

## 第四部分：方向性删除（关闭左侧/右侧）

### 4.1 delLeftViews（第 186-206 行）

```typescript
async function delLeftViews(view: TagView): Promise<DirectionalTagsViewResult> {
  const currentIndex = visitedViews.value.findIndex((v) => v.path === view.path);
  if (currentIndex === -1) {
    return { visitedViews: [...visitedViews.value] };
  }

  visitedViews.value = visitedViews.value.filter((item, index) => {
    if (index >= currentIndex || item?.affix) {
      return true;  // 保留：当前标签 + 右侧标签 + 所有 affix
    }
    const cachedIndex = cachedViews.value.indexOf(item.fullPath);
    if (cachedIndex > -1) {
      cachedViews.value.splice(cachedIndex, 1);  // 同步删缓存
    }
    return false;
  });

  return { visitedViews: [...visitedViews.value] };
}
```

逻辑：找到目标标签索引，保留 `index >= currentIndex` 的标签（目标及其右侧），同时保护所有 affix 标签。被删除的左侧标签同步从 `cachedViews` 中移除。

**示例**：标签栏为 `[Dashboard(affix), 用户管理, 角色管理, 菜单管理]`，调用 `delLeftViews(角色管理)`：

| 标签 | 索引 | `index >= 2`？ | `affix`？ | 结果 |
|------|------|:-:|:-:|------|
| Dashboard | 0 | false | true ✅ | **保留** |
| 用户管理 | 1 | false | false | **删除**+删缓存 |
| 角色管理 | 2 | true ✅ | — | **保留** |
| 菜单管理 | 3 | true ✅ | — | **保留** |

结果：`[Dashboard, 角色管理, 菜单管理]`

### 4.2 delRightViews（第 211-231 行）

```typescript
async function delRightViews(view: TagView): Promise<DirectionalTagsViewResult> {
  const currentIndex = visitedViews.value.findIndex((v) => v.path === view.path);
  if (currentIndex === -1) {
    return { visitedViews: [...visitedViews.value] };
  }

  visitedViews.value = visitedViews.value.filter((item, index) => {
    if (index <= currentIndex || item?.affix) {
      return true;  // 保留：当前标签 + 左侧标签 + 所有 affix
    }
    const cachedIndex = cachedViews.value.indexOf(item.fullPath);
    if (cachedIndex > -1) {
      cachedViews.value.splice(cachedIndex, 1);
    }
    return false;
  });

  return { visitedViews: [...visitedViews.value] };
}
```

与 delLeftViews 的唯一区别：保留条件是 `index <= currentIndex`（目标及其左侧）。

## 第五部分：工具方法

### 5.1 updateVisitedView（第 131-138 行）

```typescript
function updateVisitedView(view: TagView) {
  for (const v of visitedViews.value) {
    if (v.path === view.path) {
      Object.assign(v, view);  // 原地合并，保持响应式
      break;
    }
  }
}
```

按 `path` 匹配，`Object.assign` 原地更新属性。用于路由 query 变化后更新标签的 `fullPath`（如 `/detail?id=1` → `/detail?id=2`，path 相同但 fullPath 不同）。

### 5.2 updateTagName（第 145-151 行）

```typescript
function updateTagName(fullPath: string, title: string) {
  const tag = visitedViews.value.find((tag: TagView) => tag.fullPath === fullPath);
  if (tag) {
    tag.title = title;
  }
}
```

按 `fullPath` 查找，只更新标题。适用于动态标题场景（如"用户详情-张三"）。

### 5.3 isActive（第 266-268 行）

```typescript
function isActive(tag: TagView) {
  return tag.path === route.path;
}
```

判断标签是否为当前路由。用 `path` 而非 `fullPath` 匹配，保证 `/detail?id=1` 和 `/detail?id=2` 都对应同一个标签高亮。

### 5.4 toLastView（第 273-287 行）

```typescript
function toLastView(visitedViews: TagView[], view?: TagView) {
  const latestView = visitedViews.slice(-1)[0];
  if (latestView && latestView.fullPath) {
    router.push(latestView.fullPath);
  } else {
    if (view?.name === "Dashboard") {
      router.replace("/redirect" + view.fullPath);
    } else {
      router.push("/");
    }
  }
}
```

关闭当前标签后的跳转逻辑：
- 有剩余标签 → 跳转到列表最后一个
- 无剩余标签，关闭的是 Dashboard → `/redirect` 方式强制刷新首页
- 无剩余标签，关闭的是其他 → 跳转首页 `/`

### 5.5 closeCurrentView（第 292-308 行）

```typescript
function closeCurrentView() {
  const tags: TagView = {
    name: route.name as string,
    title: route.meta.title as string,
    path: route.path,
    fullPath: route.fullPath,
    icon: route.meta?.icon as string | undefined,
    affix: route.meta?.affix,
    keepAlive: route.meta?.keepAlive,
    query: route.query,
  };
  delView(tags).then((res) => {
    if (isActive(tags)) {
      toLastView(res.visitedViews, tags);
    }
  });
}
```

从当前 `route` 构造 TagView，调用 `delView` 删除，如果删除的是当前激活标签则自动跳转。

## 整体数据流

```
用户点击菜单 → router.push → 路由变化
  → LayoutTagsView watch(route)
    → addCurrentTag()
      → store.addView()
        → addVisitedView() → visitedViews[]（响应式，模板 v-for 重新渲染）
        → addCachedView()  → cachedViews[]（keep-alive 联动）
```

## 核心设计思想

1. **两列表分离**：`visitedViews` 管 UI 渲染，`cachedViews` 管 keep-alive 缓存
2. **删除必同步**：删 visited 的同时删对应 cached
3. **affix 始终受保护**：任何批量删除操作都不影响固定标签
4. **async 化**：相比旧版用 `new Promise(resolve => {...})`，新版统一用 `async function` 自动返回 Promise，代码更简洁，`delView`/`delOtherViews` 用 `Promise.all` 并行执行两个操作
