# LayoutTagsView 组件详解

> 文件路径：`src/layouts/components/LayoutTagsView.vue`

## 第一部分：模板结构概览

### 1.1 整体骨架

模板由三个区域组成：

```
┌─────────────────────────────────────────────────────────┐
│  [el-scrollbar]  ← 水平滚动的标签列表                    │
│    ├─ 标签1 (icon + 标题 + ×)                           │
│    ├─ 标签2 (icon + 标题 + ×)                           │
│    └─ 标签3 (icon + 标题 + ×) ← affix 无 ×             │
├─────────────────────────────────────────────────────────┤
│  操作按钮区                                             │
│    [刷新] [全屏] [▼ 下拉菜单]                            │
├─────────────────────────────────────────────────────────┤
│  右键菜单 (Teleport to body)                            │
│    刷新 / 关闭 / ─ / 关闭其它 / 关闭左侧 / 关闭右侧 / ─ / 关闭所有  │
└─────────────────────────────────────────────────────────┘
```

### 1.2 标签列表

```html
<div v-for="tag in visitedViews" :key="tag.fullPath"
  :class="{ 'is-active': tagsViewStore.isActive(tag), 'is-affix': tag.affix }"
  @click="openTag(tag)"
  @click.middle="handleMiddleClick(tag)"
  @contextmenu.prevent="openContextMenu(tag, $event)">
  <!-- 图标 -->
  <template v-if="tag.icon">
    <el-icon v-if="isEpIcon(tag.icon)" :size="14">
      <component :is="toEpIconName(tag.icon)" />
    </el-icon>
    <span v-else class="layout-tabs__item-icon" :class="`i-svg:${tag.icon}`" />
  </template>
  <!-- 标题 -->
  <span>{{ translateRouteTitle(tag.title) }}</span>
  <!-- 关闭按钮（affix 不显示） -->
  <span v-if="!tag.affix" @click.stop="closeSelectedTag(tag)">
    <div class="i-svg:close" />
  </span>
</div>
```

关键交互：
- **左键点击** → `openTag()` → `router.push`
- **中键点击** → `handleMiddleClick()` → 关闭标签
- **右键点击** → `openContextMenu()` → 显示自定义右键菜单
- **关闭按钮** → `@click.stop` 阻止冒泡，避免触发 `openTag`

### 1.3 操作按钮区

```html
<!-- 刷新按钮 -->
<button @click="refreshSelectedTag(currentTag)" aria-label="刷新当前">
  <el-icon :size="16"><Refresh /></el-icon>
</button>
<!-- 全屏按钮 -->
<button @click="appStore.toggleContentFullscreen()" aria-label="内容全屏">
  <div v-if="!appStore.contentFullscreen" class="i-svg:fullscreen" />
  <div v-else class="i-svg:fullscreen-exit" />
</button>
<!-- 下拉菜单 -->
<el-dropdown trigger="click" @command="handleActionCommand">
  <button aria-label="页签操作"><el-icon><ArrowDown /></el-icon></button>
  <template #dropdown>
    <el-dropdown-menu>
      <el-dropdown-item command="refresh">刷新当前</el-dropdown-item>
      <el-dropdown-item v-if="currentTag && !currentTag.affix" command="closeCurrent">关闭当前</el-dropdown-item>
      <el-dropdown-item divided command="closeOtherTags">关闭其它</el-dropdown-item>
      <el-dropdown-item command="closeLeftTags">关闭左侧</el-dropdown-item>
      <el-dropdown-item command="closeRightTags">关闭右侧</el-dropdown-item>
      <el-dropdown-item divided command="closeAllTags">关闭所有</el-dropdown-item>
    </el-dropdown-menu>
  </template>
</el-dropdown>
```

注意：**"关闭当前"** 只在当前标签不是 affix 时显示（`v-if="currentTag && !currentTag.affix"`）。

### 1.4 右键菜单

使用 `<Teleport to="body">` 将右键菜单渲染到 `<body>` 下，避免被父组件的 `overflow: hidden` 裁剪。

菜单项与下拉菜单类似，但多了一个基于 `isFirstView` / `isLastView` 的条件控制：
- 第一项前面没有标签时，不显示"关闭左侧"
- 最后一项时，不显示"关闭右侧"

### 2.2 ContextMenu 接口与状态

```typescript
interface ContextMenu {
  visible: boolean;
  x: number;
  y: number;

const selectedTag = ref<TagView | null>(null);

const contextMenu = reactive<ContextMenu>({
  visible: false,
```

- `selectedTag`：右键选中的标签（与当前激活标签 currentTag 不同）

### 2.3 computed 属性

#### currentTag

```typescript
const currentTag = computed(() => {
  return visitedViews.value.find((tag) => tagsViewStore.isActive(tag)) || null;
});
```

在 visitedViews 中找出与当前路由匹配的标签（通过 `isActive` 判断）。

#### tabsStyleClass

```typescript
const tabsStyleClass = computed(() => {
  switch (settingsStore.tagsViewStyle) {
    case TagsViewStyle.CARD: return "layout-tabs--card";
    case TagsViewStyle.LINE: default: return "layout-tabs--line";
  }
});
```

根据设置切换 Card 或 Line 样式。

#### contextMenuStyle

```typescript
const contextMenuStyle = computed(() => {
  // 计算菜单位置，防止溢出视口
  let x = contextMenu.x, y = contextMenu.y;
  if (x + menuWidth > window.innerWidth) x = window.innerWidth - menuWidth - padding;
  if (y + menuHeight > window.innerHeight) y = window.innerHeight - menuHeight - padding;
  return { left: `${Math.max(padding, x)}px`, top: `${Math.max(padding, y)}px` };
});
```

#### isFirstView / isLastView

```typescript
const isFirstView = computed(() => {
  // 右键菜单：判断"关闭左侧"是否可用
  return selectedTag.value?.path === "/dashboard"
    || selectedTag.value?.fullPath === visitedViews.value[1]?.fullPath;
});

const isLastView = computed(() => {
  // 右键菜单：判断"关闭右侧"是否可用
  return selectedTag.value?.fullPath === visitedViews.value[visitedViews.value.length - 1]?.fullPath;
});
```

#### routePathMap

```typescript
const routePathMap = computed(() => {
  const map = new Map<string, TagView>();
  visitedViews.value.forEach((tag) => map.set(tag.path, tag));
  return map;
});
```

从 `path → TagView` 的映射，用于 `updateCurrentTag` 中快速查找。

### 2.4 图标工具函数

```typescript
const isEpIcon = (icon: string) => icon.startsWith("el-icon");

const toEpIconName = (icon: string) =>
  icon.replace("el-icon-", "").replace(/(^|-)\w/g, (s) => s.slice(-1).toUpperCase());
```

处理两种图标格式：
- Element Plus 图标（`el-icon-xxx`）→ 渲染为动态组件
- 自定义 SVG 图标 → 渲染为 CSS class

## 第三部分：标签初始化与路由监听

### 3.1 affix 标签的初始化

```typescript
const extractAffixTags = (routes: RouteRecordRaw[], basePath = "/"): TagView[] => {
  const affixTags: TagView[] = [];
  const traverse = (routeList: RouteRecordRaw[], currentBasePath: string) => {
    routeList.forEach((route) => {
      const fullPath = resolve(currentBasePath, route.path);
      if (route.meta?.affix) {
        affixTags.push({ path: fullPath, fullPath, name: String(route.name || ""),
          title: route.meta.title, icon: route.meta.icon, affix: true, keepAlive: ... });
      }
      if (route.children?.length) traverse(route.children, fullPath);
    });
  };
  traverse(routes, basePath);
  return affixTags;
};

const initAffixTags = () => {
  const affixTags = extractAffixTags(permissionStore.routes);
  affixTags.forEach((tag) => { if (tag.name) tagsViewStore.addVisitedView(tag); });
};
```

递归遍历所有路由配置，找出 `meta.affix: true` 的路由（如 Dashboard），在组件挂载时添加。

### 3.2 路由监听

```typescript
watch(
  route,
  () => {
    addCurrentTag();
    updateCurrentTag();
  },
  { immediate: true }
);
```

核心入口：路由变化时执行两个操作。

### 3.3 addCurrentTag

```typescript
const addCurrentTag = () => {
  if (!route.meta?.title) return;
  if (isExternal(route.path) || isExternal(route.fullPath)) return;
  tagsViewStore.addView({
    name: route.name as string,
    title: route.meta.title,
    path: route.path,
    fullPath: route.fullPath,
    icon: route.meta.icon,
    affix: route.meta.affix || false,
    keepAlive: route.meta.keepAlive || false,
    query: route.query,
  });
};
```

将当前路由信息封装成 TagView 调用 `store.addView()`，这是**整个标签系统的触发入口**。

### 3.4 updateCurrentTag

```typescript
const updateCurrentTag = () => {
  nextTick(() => {
    const currentTag = routePathMap.value.get(route.path);
    if (currentTag && currentTag.fullPath !== route.fullPath) {
      tagsViewStore.updateVisitedView({ ... });
    }
  });
};
```

当 query 参数变化（`/detail?id=1` → `/detail?id=2`）时，更新标签的 `fullPath`。

### 3.5 组件挂载

```typescript
onMounted(() => { initAffixTags(); });
useContextMenuManager();
```

## 第四部分：标签交互操作

### 4.1 openTag（第 308-318 行）

```typescript
const openTag = (tag: TagView) => {
  if (isExternal(tag.fullPath)) {
    window.open(tag.fullPath, "_blank", "noopener,noreferrer");
    return;
  }
  router.push({ path: tag.fullPath, query: tag.query });
};
```

点击标签时路由跳转，外部链接则新窗口打开。

### 4.2 closeSelectedTag（第 381-389 行）

```typescript
const closeSelectedTag = (tag: TagView | null) => {
  if (!tag) return;
  tagsViewStore.delView(tag).then((result) => {
    if (tagsViewStore.isActive(tag)) {
      tagsViewStore.toLastView(result.visitedViews, tag);
    }
  });
};
```

关闭标签，如果关闭的是当前激活标签则自动跳转。

### 4.3 refreshSelectedTag（第 372-379 行）

```typescript
const refreshSelectedTag = (tag: TagView | null) => {
  if (!tag) return;
  tagsViewStore.delCachedView(tag);
  nextTick(() => {
    router.replace("/redirect" + tag.fullPath);
  });
};
```

刷新：先从缓存中移除，然后跳转到 `/redirect` 路由触发组件重新创建。

### 4.4 handleMiddleClick（第 339-343 行）

```typescript
const handleMiddleClick = (tag: TagView) => {
  if (!tag.affix) closeSelectedTag(tag);
};
```

鼠标中键点击标签关闭。

### 4.5 handleScroll（第 356-370 行）

```typescript
const handleScroll = (event: WheelEvent) => {
  closeContextMenu();

  if (!scrollWrapper) return;
```

## 第五部分：批量关闭与菜单管理

### 5.1 closeDirectionalTags（第 394-407 行）

```typescript
function closeDirectionalTags(
  source: Ref<TagView | null>,
  batchFn: (tag: TagView) => Promise<{ visitedViews: TagView[] }>
) {
  const tag = source.value;
  if (!tag) return;
  batchFn(tag).then(({ visitedViews }) => {
    const stillVisible = visitedViews.some((v) => v.path === route.path);
    if (!stillVisible) tagsViewStore.toLastView(visitedViews);
  });
}
```

通用函数，接收标签 ref 和 store 方法，删除后检查当前路由是否被删除，是则跳转。

四种调用方式：
```typescript
const closeLeftTagsForActive  = () => closeDirectionalTags(currentTag,   tagsViewStore.delLeftViews);
const closeRightTagsForActive = () => closeDirectionalTags(currentTag,   tagsViewStore.delRightViews);
const closeLeftTags           = () => closeDirectionalTags(selectedTag,  tagsViewStore.delLeftViews);
const closeRightTags          = () => closeDirectionalTags(selectedTag,  tagsViewStore.delRightViews);
```

`ForActive` 后缀来自**下拉菜单**（对当前标签操作），无后缀来自**右键菜单**（对右键选中标签操作）。

### 5.2 closeOtherTags 和 closeOtherTagsForActive

```typescript
// 下拉菜单：关闭其他（保留当前激活标签）
const closeOtherTagsForActive = () => {
  if (!currentTag.value) return;
  tagsViewStore.delOtherViews(currentTag.value).then(() => updateCurrentTag());
};

// 右键菜单：关闭其他（保留右键选中标签）
const closeOtherTags = () => {
  if (!selectedTag.value) return;
  router.push(selectedTag.value);   // 先跳转到选中标签
  tagsViewStore.delOtherViews(selectedTag.value).then(() => updateCurrentTag());
};
```

注意 `closeOtherTags` 会先 `router.push(selectedTag.value)` — 因为右键选中的可能不是当前页，需要先跳转过去。

### 5.3 closeAllTags（第 432-436 行）

```typescript
const closeAllTags = (tag: TagView | null) => {
  tagsViewStore.delAllViews().then((result) => {
    tagsViewStore.toLastView(result.visitedViews, tag || undefined);
  });
};
```

删除所有（保留 affix），然后跳转到结果中的最后一个（通常是 Dashboard）。

### 5.4 handleActionCommand（第 209-230 行）

```typescript
const handleActionCommand = (command: string) => {
  switch (command) {
    case "refresh":        refreshSelectedTag(currentTag.value); break;
    case "closeCurrent":   closeSelectedTag(currentTag.value); break;
    case "closeOtherTags": closeOtherTagsForActive(); break;
    case "closeLeftTags":  closeLeftTagsForActive(); break;
    case "closeRightTags": closeRightTagsForActive(); break;
    case "closeAllTags":   closeAllTags(currentTag.value); break;
  }
};
```

下拉菜单命令的分发器。

### 5.5 右键菜单管理（第 438-454 行）

```typescript
const useContextMenuManager = () => {
  const handleOutsideClick = () => closeContextMenu();
  watchEffect(() => {
    if (contextMenu.visible) {
      document.addEventListener("click", handleOutsideClick);
    } else {
      document.removeEventListener("click", handleOutsideClick);
    }
  });
  onBeforeUnmount(() => document.removeEventListener("click", handleOutsideClick));
};
```

用 `watchEffect` 响应式监听菜单显隐，显示时注册全局点击关闭，隐藏时移除监听。

## 整体数据流

```
初始化:
  onMounted → initAffixTags() → 遍历路由 → addVisitedView(affix标签)

路由变化:
  watch(route) → addCurrentTag() → store.addView() → visitedViews 自动更新 UI
               → updateCurrentTag() → query 变化时更新 fullPath

关闭操作:
  closeSelectedTag → store.delView() → 如果关闭的是当前页 → toLastView()
  refreshSelectedTag → store.delCachedView() → /redirect → 路由重走 → addCurrentTag()
```
